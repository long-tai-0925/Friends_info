# Friends_info
讓好友們更加方便的互相分享對方個人網頁

# 簡介：
user可以勾選想要加誰
提交勾選完的選項後
可以發email問被勾選的user同不同意

然後這個平台可以給user_api
這樣對方同意就直接加進去
不用user手動更新自己的頁面

# 使用者Emali：
## A 請求 B 加入 A 網頁時
### A：
```
您邀請已送出下列使用者成為你個人網頁的好友清單中的一員
目前邀請Email已寄出，請慨心等候
您邀請的使用者：
{user_name_list}
```
### Ｂ：
```
您被邀請成為 {A_user_name} 個人網頁好友清單中的一員
這是對方的網頁 {A_user_links}
請問你是否同意？
[同意]
如需拒絕則無須理會此郵件 
```

## A 請求 B 將 A 加入 B 網頁時
### A：
```
您邀請成為 {B_user_name} 個人網頁好友清單中的一員
目前邀請Email已寄出，請慨心等候
```
### B：
```
{A_user_name} 希望成為您個人網頁好友清單中的一員
這是對方的網頁 {A_user_links}
請問你是否同意？
[同意]
如需拒絕則無須理會此郵件 
```

## 當新的使用者加入時
### 勾選通知的使用者：
```
哈摟！有新的使用者加入了，您是否要將其加入自己的個人網頁{user_links}
[是]
如需否則無須理會此郵件 
```

## Email 中的回應：
### A 請求 B 加入 A 網頁時，B 點擊同意：
```B 增加到 A 的好友清單```
### A 請求 B 將 A 加入 B 網頁時，B 點擊同意：
```A 增加到 B 的好友清單```
### 勾選通知的使用者，選擇是：
```通知新的使用者 "A 請求 B 加入 A 網頁時" 類型的Email，新使用者為 B```
```通知 A 使用者 "A 請求 B 加入 A 網頁時" 類型的Email，使用者為 A```
